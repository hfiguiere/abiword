Warum AbiWord?

Das 'Single Document Interface'

AbiSource, Inc.

AbiWord ist ein kleines und einfaches Satzprogramm, welches schnell und komplett ist. Es ist ein Teil des AbiSource Project, welches von der SourceGear company gesponsort ist. Das Ziel dieses Projektes ist die Entwicklung eines plattformübergreifenden Open-Source-Office-Paketes. Den Beginn dieses Projekts stellt das Textverarbeitungsprogramm AbiWord dar. Abiwords Features beinhalten " look & feel " (siehe diesen Screenshot), die Möglichkeit, Seite und Abschnitte zu formatieren, Rechtschreibprüfung, interaktive Regeln, die Integration von Dokumentstilen, unbegrenzte undo/redo Kapazität, eine Suche-und-ersetze-Funktion und das Einfügen von Bildern. Es ist ebenso möglich, Dokumeten im Microsoft Word 97 und rtf (Rich Text Format) zu importieren und Dokumente im Internet HTML-Format abzuspeichern. Verschiedene andere Merkmale sind für zukünftige Versionen geplant. Einige davon sind das Einfügen von Tabellen, Listen und das Exportieren im Microsoft Word Format. Weitere Informationen finden sich in unserer 'roadmap'.

Der wichtigste Unterschied zwischen AbiWord und den meisten anderen verfügbaren Textverarbeitungsprogrammen ist das Format der verwendeten Dateien. Ein Dokument wird in AbiWord standardmäßig mit  der Erweiterung *abw im XML- und damit im ASCII-Format abgespeichert. Diese Dateien können mit jedem beliebigen Texteditor bearbeitet werden. Dies bedeutet nicht, daß AbWord ein XML-Editor ist. AbiWord ist ein Textverarbeituntsprogramm, welches XML wie eine praktische Syntax für die Darstellung des Dokumentes verwendet. Dies ist verglichen mit den anderne traditionellen Textverarbeitungsprogrammen ein völlig neues Konzept. Es bedeutet, daß Dateien, die mit AbiWord abgespeichert wurden, nicht speziell an dieses Programm gebunden sind. Textverarbeitungsprogramme veralten in der Regel innerhalb von 5 Jahren. Die Daten sind jedoch auch dann nicht verloren.

Zuletzt ist AbiWord ein vollkommen frei und plattformübergreifend. Es funktioniert unter Windows und dem Großteil der UNIX-Systeme.

  • BeOS / INTEL (experimental)
  • BeOS / PPC (experimental)
  • Linux / Alpha
  • Linux / INTEL
  • Linux / Power PC
  • Linux / INTEL IA64 (experimental)
  • NetBSD
  • FreeBSD / INTEL
  • AIX
  • Solaris
  • Windows 95, 98, 2000, ME und NT
  • Andere
In den nächsten Jahren wird Datenverarbeitung ein großer Teil der Entwicklung verschiedener Betriebssysteme (OS) werden. AbiWord wird unter diesen speziellen OS funktionieren. Und auch hier wird die Wahl des XML-Formates sicherstellen, daß Dateien von einem Betriebssystem auf das andere übertragen werden könne. Dies ist bei herkömmlichen Programmen nicht der Fall.

Das Open Source Koncept

Die Entwicklung von Open Source Programmen unterscheidet sich von der traditionellen kommerziellen Softwareentwicklung. Der Quelltext (die Programmzeilen, die die Software selber definieren) sind frei verfügbar ebenso wie die Software selber. Für den Großteil der Benutzer ist dies weniger von Bedeutung, jedoch profitieren alle von der unaufhörlichen Verbesserungen, die Freiwillige in das Produkt einbringen. Jedermann kann an der Weiterentwicklung des Produktes teilnehmen. Dieses Konzept ist das Gegenteil der proprietären Softwareentwicklung. Dort ist die Software von einer Lizenz geschützt, die den Zugang zum Quelltext und jegliche Veränderung des Quelltextes untersagt. Das Kopieren und die Verbreitung des Programms ist ebenfalls untersagt. Weitere Informationen zum Open Source Konzept findet sich unter  www.opensource.org.
AbiWord ist daher Open Source Software. Es ist keine Shareware und es sind keine gesetzlichen, moralischen oder sonstigen Einschränkungen vorhanden. Es ist vollkommen frei. Wir wollem, daß alle diese Software frei benutzen können. Sie können sie sogar an alle Personen, sie daran interessiert sein können weitergeben.

AbiWord ist aus einem Entwicklungsmodell geboren, daß viele Leute ignorieren. Das Ergebnis der Verteilung  vieler Entwickler auf der ganzen Welt ist, daß regelmäßige Updates erscheinen. Der Benutzer von AbiWord sollte hin und wieder schauen, ob neue Programmversionen erschienen sind und daher regelmäßig auf der AbiSource Seite sicherstellen, daß er die neueste Version von AbiWord verwendet.
