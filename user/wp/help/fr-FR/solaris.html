Installation d'AbiWord sur Solaris

Bienvenue dans la communauté des utilisateurs d'AbiWord!

La dernière version d'AbiWord doit correctement s'installer et fonctionner sur les systèmes Solaris 7 avec processeur Sparc32. AbiWord est disponible à la fois au format natif Solaris "pkg" et au format gzipped tar.

L'installation du fichier au format natif Solaris "pkg" est assez simple et sera familière aux utilisateurs de Solaris.

  1. Télécharger l'archive (abisuite.x.x.x.sparc32.SunOS.5.5.1.pkg.tar.gz).
  2. Copiez l'archive dans un dossier accessible par vous et ouvrez l'archive: 
      gunzip -c abisuite.x.x.x.sparc32.SunOS.5.5.1.pkg.tar.gz | tar xvf -

    ... ou, si votre commande tar accepte les tags 'z':

      tar xvzf abisuite.x.x.x.sparc32.SunOS.5.5.1.pkg.tar.gz

    Les deux commandes créeront un dossier abisuite.

  3. En tant que superviseur, taper la commande: 
      pkgadd -d . abisuite
  4. Par défaut, Cela installera AbiWord dans le dossier /usr/local/AbiSuite de votre disque dur. Vous pouvez changer le dossier par défaut si souhaité.
  5. Vous pourrez également créer les liens vers les exécutables dans un dossier bin . Par défaut ce lien est /usr/local/bin.
  6. Merci de consulter le fichier README pour des informations complémentaires.

Le processus d'installation de l'archive Tar est très simple.

  1. Télécharger l'archive (abisuite-x.x.x-SunOS_sparc32_dynamic.tar.gz).
  2. Copiez l'archive dans un dossier accessible par vous et ouvrez l'archive: 
      gunzip -c abisuite-x.x.x-SunOS_sparc32_dynamic.tar.gz | tar xvf -

    ... ou, si votre commande tar accepte les tags 'z':

      tar xvzf abisuite-x.x.x-SunOS_sparc32_dynamic.tar.gz

    Les deux commandes créeront un dossier abisuite-x.x.x-SunOS_sparc32_dynamic.

  3. Allez dans ce dossier en tant que superviseur.
  4. Lancez le script d'installation: 
      ./install.sh
  5. Par défaut, Cela installera AbiWord dans le dossier /usr/local/AbiSuite de votre disque dur. Vous pouvez changer le dossier par défaut si souhaité.
  6. Vous pourrez également créer les liens vers les exécutables dans un dossier bin . Par défaut ce lien est /usr/local/bin.
  7. Merci de consulter le fichier README pour des informations complémentaires.
